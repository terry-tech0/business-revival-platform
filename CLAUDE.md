# CLAUDE.md — プロジェクト定義書

## 1. プロジェクト概要

- **名称:** 人財駆動型・企業再興プラットフォーム
- **コンセプト:** 10年以上停滞している成熟企業をターゲットに、社内に埋もれた人財を最適にマッチングさせ、データ（AI分析）に基づいた新規事業を創出することで企業をアップデートする。
- **種別:** 学習用プロジェクト（4人チーム開発）

## 2. アプリのフロー

```
① 市場・ターゲット入力（テキストボックス）
       ↓
② 市場分析（PEST分析 + 5つの力分析）を AI が実行
       ↓
③ AIが課題を10個生成 → 納得するまで何度でも再生成可能
       ↓  ユーザーが1つ選択
④ AIが解決策を10個生成（5軸スコアリング付き） → 納得するまで再生成可能
       ↓  ユーザーが1つ選択
⑤ 選んだ解決策に最適な4人チームを社内DBから選抜して表示
```

## 6. チーム役割分担（4人）

### 🔌 AI API担当（参考アプリの「クローラー担当」に相当）
- **担当ファイル:** `ai_engine.py`, `prompts.py`
- **役割:** OpenAI APIを呼び出し、外部から知識を取得する。参考アプリの crawler.py がWebからデータを取得するのと同様に、このアプリではAI APIからデータを取得する。
- **参考:** 元アプリの `crawler.py` の構造（fetch → parse → return）

### 🎨 フロント担当
- **担当ファイル:** `app.py`
- **役割:** Streamlit でユーザーインターフェースを構築。ステップ1〜5の画面遷移とユーザー操作を管理する。
- **参考:** 元アプリの `app.py` の構造（タブ管理、session_state）

### 🗄️ DB担当
- **担当ファイル:** `schema.sql`, `seed_data.py`, `database.py`
- **役割:** SQLiteデータベースの設計・構築・操作。人財データの管理とセッションログの保存。
- **参考:** 元アプリの `database.py`, `schema.sql` の構造

### 📊 ランキング担当
- **担当ファイル:** `matching.py`
- **役割:** 解決策のスコアリング・ランキングロジックと、人財マッチングの制御。MBTIバランス分析。
- **参考:** 元アプリの `ranking.py` の構造（スコア計算、ソート、表示用フォーマット）

## 7. セットアップ手順

```bash
# 1. 依存パッケージをインストール
pip install -r requirements.txt

# 2. ダミーデータを投入
python seed_data.py

# 3. アプリを起動
streamlit run app.py
```

## 8. スコアリング5軸

| 軸 | 評価観点 |
|---|---------|
| market_size | 市場規模・成長性 — 市場が大きく成長余地があるか |
| feasibility | 実現可能性 — 技術的に実現可能で着手できるか |
| profitability | 収益性 — 持続的な収益モデルを構築できるか |
| innovativeness | 革新性 — 既存にない新しい価値を提供できるか |
| sustainability | 持続的優位性 — 競合が簡単に真似できない優位性があるか |

## 9. 開発ルール

- **ブランチ運用:** `main` ブランチは保護。各担当は `feature/xxx` ブランチで作業し PR を出す。
- **コミットメッセージ:** `[担当名] 変更内容` 形式（例: `[DB] schema.sql に index 追加`）
- **コーディング規約:** Python は PEP 8 準拠。日本語コメント推奨。
- **環境変数:** `.env` ファイルは絶対にGitにコミットしない。
